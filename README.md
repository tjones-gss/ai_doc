# Software Modernization AI Guide

Living Docusaurus documentation for AI-assisted software modernization at Global Shop
Solutions.

The site is the source of truth for how GSS teams use AI tools in day-to-day engineering:
Cursor, ChatGPT, Codex, Augment AI, Claude Code, MCP servers, LaunchPad-hosted tools,
COBOL workflows, testing practices, prompts, and troubleshooting notes.

## Published site

The production site is configured for GitHub Pages:

- URL: <https://tjones-gss.github.io/ai_doc/>
- Repository: <https://github.com/tjones-gss/ai_doc>
- Docusaurus config: [`docusaurus.config.js`](docusaurus.config.js)

## Quick start

### Prerequisites

- Node.js 22 or newer
- npm

### Install dependencies

```bash
npm install
```

### Start local development

```bash
npm start
```

The dev server opens the site at <http://localhost:3000> with hot reload enabled.

### Build production assets

```bash
npm run build
```

Docusaurus writes the static site to `build/`.

### Serve a production build locally

```bash
npm run serve
```

## Documentation workflow

1. Edit or add Markdown under `docs/`.
2. Keep filenames kebab-case unless preserving an existing external tool name.
3. Include or update front matter, especially `title`, `description`, `last_updated`,
   and relevant `tags`.
4. Add sidebar entries in `sidebars.js` when the page is not covered by an autogenerated
   catalog directory.
5. Store served assets in `static/`; keep source-only images or design files in `images/`
   if needed.
6. Run focused validation before opening a pull request.

## Content standards

- Write for engineers and adjacent technical readers who need practical, repeatable
  guidance.
- Prefer concrete workflows, command examples, and links to related internal docs over
  broad tool descriptions.
- Update `last_updated` whenever a published doc changes.
- Keep shared styling in `src/css/custom.css` instead of inline styles.
- For COBOL guidance, treat `C:\COBOL\` as the source-of-truth code location.

## Deployment

GitHub Pages deployment uses the Docusaurus settings in `docusaurus.config.js`:

- `url`: `https://tjones-gss.github.io`
- `baseUrl`: `/ai_doc/`
- `organizationName`: `tjones-gss`
- `projectName`: `ai_doc`

To deploy manually:

```bash
npm run deploy
```

CI should build the site and publish the generated `build/` artifact to GitHub Pages.

## Troubleshooting

- If local docs look stale, run `npm run clear` and restart `npm start`.
- If a build fails after moving docs, check `sidebars.js` and broken links.
- If Playwright tests fail because browsers are missing, install them with
  `npx playwright install`.
